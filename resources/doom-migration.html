1. First Impressions

Started with a clean install since my .emacs file from windows wasn’t working right and I was tired of fighting it. In my normal workflow of typing in org files I’ve noticed that it does something that I would have preferred out of the box with emacs on windows (and eventually tried to do in one way or another).

  1. auto-indentation doesn’t suck
  2. Spellcheck works and offers auto-completions which is NICE
  3. when I created a new org file, it pasted the #+title in the first line for me. (note to self, find this and add more to it)

2. Config Files

From the main page of doom emacs

You can configure Doom by tweaking the files found in your DOOMDIR. Doom expects this directory to be found in one of:

~/.config/doom (respects $XDGCONFIGHOME) or ~/.doom.d

2.1. doom config files

if we look in the ~/.doom.d we find the trifecta of doom config files. 

ls ~/.doom.d
ls ~/.config/doom

config.el init.el packages.el

ls: cannot access ’home/rbrins.config/doom’: No such file or directory

2.2. or ~/.config files

I’ll use the ~/.doom.d directory for now but eventually want to start storing my config files in ~/.config/ as I get more use to linux as my daily driver. Either way, according to the config information page, init.el is for modules, config.el is for the private customization, and packages.el is for installing the packages. It explicitly mentions not using the customize in default which is nice considering I was fighting that so often.

3. Preconfigured Canvas

I saw a reddit comment that mentioned that vanilla emacs was a blank canvas, spacemacs was the canvas painted by the community and ever growing, and doom emacs was a canvas with a little started on the canvas and currated tools. This is very apt for what I’ve seen from doom so far and enjoy it.

This has helped me adopt org-agenda and org-roam much faster than I was adopting before since it has the bindings and auto-completion already existing for these packages.
